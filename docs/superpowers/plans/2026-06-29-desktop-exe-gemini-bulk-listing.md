# Desktop EXE Gemini Bulk Listing Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a Windows desktop entry point that lets operators configure Gemini and Noon credentials, paste bulk 1688 URLs, import Excel URL batches, run the existing listing pipeline, and package the app as an EXE.

**Architecture:** Keep the existing CLI pipeline as the source of truth. Add a small app service layer for local settings, Gemini native generation, URL parsing, batch job execution, and a PySide6 GUI that calls those services without duplicating pipeline logic.

**Tech Stack:** Python 3.11+, standard library `unittest`, Tkinter for GUI, PyInstaller for EXE packaging, existing `openpyxl`, `pillow`, `lxml`, `cryptography`.

## Global Constraints

- Never commit real API keys, Noon credentials, cookies, generated runs, or local config files.
- The operator only fills API keys and file paths; provider URLs and request bodies are generated by the app.
- First stability target is bulk pasted 1688 URLs. Excel import is supported through the same batch service.
- GUI must remain usable if AI is not configured; fallback rule-based listing generation must still work.
- Live Noon submission remains explicit and off by default.
